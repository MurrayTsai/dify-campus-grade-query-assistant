## 核心流程

### 1. 用户输入

用户用自然语言提出查询需求，例如：

```text
查询全校各科目平均分情况
```

### 2. LLM 生成 SQL

第一个 LLM 节点根据用户问题生成 JSON 格式的 SQL 数组。

### 3. 解析 SQL

代码节点从模型输出中提取 JSON，并将 SQL 数组传入迭代节点。

### 4. 循环执行 SQL

迭代节点逐条执行 SQL，每条 SQL 交给 Database 插件执行。

### 5. 汇总查询结果

代码节点将所有 SQL 执行结果合并，传给第二个 LLM 节点。

### 6. LLM 生成分析结果

第二个 LLM 节点根据用户问题、SQL 语句和查询结果生成 Markdown 形式的回答，并判断是否需要 ECharts 图表。

### 7. 图表分支

条件分支根据 `chartType` 字段选择：

- 线性图
- 柱状图
- 饼图
- 直接文本回复

## 当前可优化点

1. SQL 生成节点应明确限制只允许 `SELECT` 查询。
2. Prompt 中“只包含 sql 字段”和示例中的 `title + sql` 存在轻微不一致，建议统一为 `title + sql`。
3. 对学生成绩查询应增加权限控制说明，避免查询敏感个人成绩。
4. JSON 解析代码可以增强容错，兼容模型没有输出代码块的情况。
5. 图表数据格式可以进一步规范，减少 ECharts 节点解析失败概率。
